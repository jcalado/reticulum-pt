O que é o Reticulum?

Uma ferramenta para construir milhares de redes — sem kill-switches, sem vigilância, sem censura. Redes para seres humanos.

Criptografia por defeito

Toda a comunicação é encriptada com criptografia de curva elíptica e AES. Não é possível enviar pacotes sem encriptação na rede.

Agnóstico ao meio

Funciona sobre LoRa, packet radio, WiFi, Ethernet, ligações série, links ópticos e praticamente qualquer meio com 5 bps e 500 bytes de MTU.

Sem endereços de origem

Nenhum pacote contém informação sobre o endereço, local, máquina ou pessoa de onde originou. Anonimato do iniciador por design.

Auto-configurável

A rede forma-se automaticamente. Sem necessidade de configurar endereços IP, sub-redes, tabelas de roteamento ou hierarquias.

Endereços portáteis

Os endereços são soberanos e portáteis. Uma vez criados, podem mover-se fisicamente para outro ponto da rede e continuar acessíveis.

Forward Secrecy

Chaves efémeras e troca de chaves ECDH em Curve25519 garantem forward secrecy por defeito em todas as ligações.

Como funciona?

Do zero à tua primeira rede mesh encriptada em minutos.

01

Instalar o Reticulum

Uma única linha de comando. Funciona em Linux, macOS, Windows, Raspberry Pi e Android.

02

Ligar interfaces

Configura rádios LoRa, WiFi, Ethernet, packet radio ou qualquer combinação de meios disponíveis.

03

Mesh automática

O Reticulum trata do resto. Dispositivos em qualquer meio podem comunicar entre si automaticamente.

04

Comunicar

Usa o Sideband, Nomad Network ou MeshChat para mensagens, chamadas de voz e transferência de ficheiros.

# Instalação rápida
pip install rns

# Iniciar o daemon
rnsd

# Ver estado da rede
rnstatus

# Explorar nós na testnet
rnprobe <destination_hash>

Configurações de Interface

Exemplos de configuração para o ficheiro ~/.reticulum/config. Escolhe a interface adequada ao teu setup.

# Interface RNode LoRa — 868 MHz (Europa / Portugal)

  [[RNode LoRa Interface]]
    type = RNodeInterface
    port = /dev/ttyUSB0
    frequency = 869556250
    bandwidth = 62500
    txpower = 25
    spreadingfactor = 7
    codingrate = 5
frequency 869.556 MHz Sub-banda g3 (869.4–869.65 MHz) — até 500 mW ERP e 10% duty cycle
bandwidth 62.5 kHz Largura de banda do canal — bom equilíbrio entre alcance e débito
txpower 25 dBm Potência de transmissão (~316 mW) — dentro do limite de 500 mW
spreadingfactor 7 Fator de espalhamento mais rápido — maior débito, menor alcance vs SF12
codingrate 5 Menor redundância FEC — maximiza a taxa de dados útil 
# Interface RNode LoRa — 433 MHz (ISM / Portugal)

  [[RNode LoRa 433]]
    type = RNodeInterface
    port = /dev/ttyUSB0
    frequency = 433775000
    bandwidth = 62500
    txpower = 7
    spreadingfactor = 7
    codingrate = 5
frequency 433.775 MHz Banda ISM 433 MHz — ANACOM permite até 10 mW e 10% duty cycle
bandwidth 62.5 kHz Largura de banda do canal — bom equilíbrio entre alcance e débito
txpower 7 dBm Potência de transmissão (~5 mW) — dentro do limite ANACOM de 10 mW
spreadingfactor SF7 Fator de espalhamento mais rápido — maior débito, menor alcance vs SF12
codingrate 4/5 Menor redundância FEC — maximiza a taxa de dados útil 
# Interface RNode LoRa — 2.4 GHz

  [[RNode LoRa 2.4G]]
    type = RNodeInterface
    port = /dev/ttyUSB0
    frequency = 2475000000
    bandwidth = 812000
    txpower = 20
    spreadingfactor = 8
    codingrate = 5
frequency 2475 MHz Banda ISM 2.4 GHz — sem restrições de duty cycle
bandwidth 812 kHz Largura de banda elevada — maior débito que LoRa sub-GHz
txpower 20 dBm Potência de transmissão (~100 mW)
spreadingfactor 8 Bom equilíbrio entre débito e alcance para 2.4 GHz
codingrate 5 Menor redundância FEC — maximiza a taxa de dados útil 
# WiFi 2.4 GHz — 802.11 B/G/N (RNode AP)

  Canal        = 8
  Normas       = 802.11 B/G/N
  TX Power     = 20 dBm
  SSID         = RNode
  Password     = R3ticulumPT#
  Encriptação  = WPA/WPA2
Canal 8 Canal WiFi 2.4 GHz (2447 MHz)
SSID RNode Nome da rede WiFi do RNode em modo AP
Password R3ticulumPT# Password da rede — WPA/WPA2
TX Power 20 dBm Potência de transmissão WiFi (~100 mW) 
# Interface Auto — WiFi / Ethernet (zero config)

  [[Default Interface]]
    type = AutoInterface
    enabled = yes
type AutoInterface Descoberta automática de nós Reticulum na rede local via IPv6 multicast
enabled yes Sem configuração adicional — funciona em qualquer rede WiFi ou Ethernet 
# TCP Client — Rede Reticulum Portugal

  [[Reticulum PT]]
    type = TCPClientInterface
    enabled = yes
    target_host = network.reticulum.pt
    target_port = 4242
target_host network.reticulum.pt Servidor da rede Reticulum em Portugal
target_port 4242 Porta TCP do servidor — ligação automática com reconexão

Estamos a construir uma rede de nós Reticulum em Portugal. Com hardware a partir de ~15€ e alguma curiosidade, podes fazer parte de uma rede de comunicação resiliente e soberana.
